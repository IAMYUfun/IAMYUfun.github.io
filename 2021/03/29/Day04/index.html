面向对象

什么是面向对象

  • 面向对象编程（OPP,Object-Oriented Programming）
  • 面向对象编程的本质：以类的方式组织代码，以对象封装数据。
  • 抽象
  • 三大特性：
    • 封装
    • 继承
    • 多态
  • 从代码运行角度考虑先有类后有对象。类是对象的模板。

方法回顾

  • 方法定义
    • 修饰符
    • 返回类型
    • break：跳出switch，结束循环 和 return 的区别
    • 方法名：注意规范，见名知意
    • 参数列表：（参数类型，参数名）
    • 异常抛出
  • 方法的调用：递归
    • 静态方法
    • 非静态方法
    • 形参和实参
    • 值传递（Java）和引用传递
    • this关键字

创建与初始化对象

  • 使用new关键字创建对象
  • 使用new关键字创建的时候，除了分配内存空间之外，还会给创建好的对象进行默认的初始化以及对类中构造器的调用。
  • 构造器
    • 必须和类的名字相同
    • 没有返回值，也不能写void
  • 构造器必须掌握

封装

  • 该露的露，该藏的藏
    • 高内聚，低耦合。高内聚就是类内部数据操作细节自己完成，不允许外部干涉，低耦合，仅暴露少量方法
  • ==属性私有，get/set==
  • 提供一些public的get，set方法
  • alter + insert 自动生成get，set方法（安全判断）

封装的意义

  1. 提高程序的安全性，保护数据
  2. 隐藏代码的实现细节
  3. 统一接口
  4. 系统的可维护性增加，便于代码重构

继承

super

super注意点：

  1. super调用父类的构造方法，必须在构造方法的第一个
  2. super必须只能出现在子类的方法或者构造方法中
  3. super和this不能同时调用构造方法

Vs this：

  1. 代表的对象不同：
    • this：本身调用者这个对象
    • super：代表父类对象的应用
  2. 前提：
    • this：没有继承也可使用
    • super：有继承才可使用
  3. 构造方法：
    • this（）；本类的构造
    • super（）；父类的构造

方法重写

重写都是方法的重写，跟属性无关

父类的引用指向子类

静态方法

不能重写

B a = new A（）；

如果父类和子类有相同的静态方法，调用哪一个，与创建对象时，左边的类型有关。

非静态方法

可重写（私有方法不能重写）

要有继承关系，子类重写父类方法

  1. 方法名必须相同
  2. 参数列表必须相同
  3. 修饰符，范围可扩大但不能缩小
  4. 抛出异常，范围可以被缩小但不能扩大

为什么需要重写：

  1. 父类的功能，子类不一定需要或满足。
  2. alt + insert ：override

多态

  • 动态编译：类型：可扩展性
  • Student s1 = new Student()； Person s2 = new Student()；
  • 如果方法被重写，则s1，s2，均执行子类的方法
  • s1能执行非继承方法，s2不能执行非继承方法
  • 接口的实现和类的继承是相同的。
  • 向上转型是自动发生的，如果参数类型为Person ，s1会自动转为Person类型，不能再使用非继承方法。

多态注意事项：

  1. 多态是方法的多态，属性没有多态
  2. 父类和子类，有联系 类型转换异常 classCastException
  3. 存在条件：继承关系，方法需要重写，父类引用指向子类对象

static 方法属于类，不属于实例

final 常量（断子绝孙）

private 方法

不能重写，没有多态

强制类型转换

instanceof

object instanceof Student 返回布尔值

判断对象的类型是否相似

高转低（父类转子类）向下转型，需要强制类型转换，可能丢失方法。

子类转换为父类，向上转型

抽象类

abstract

约束，有人帮我们实现

extends

abstract 抽象方法只有方法名，没有方法实现。

抽象类的所有方法，继承它的子类，要实现方法

  1. 抽象类不能new 只能靠子类去实现它
  2. 抽象类可以写普通方法
  3. 抽象方法必须在抽象类中
  4. 提高开发效率

接口

extends只能单继承

可实现多继承

  • 普通类：只有具体实现
  • 抽象类：具体实现和规范（抽象方法）都有
  • 接口：只有规范，自己无法写方法~专业的约束 ，约束和实现分离：面向接口编程。
  • 接口的本质是契约

interface

接口中所有定义默认都是抽象的public abstract（可省略不写）

类实现接口 implements 接口

alt + insert + implements

接口作用：

  1. 约束
  2. 定义一些方法，让不同的人实现
  3. public abstract
  4. public static final
  5. 接口不能被实例化；没有构造方法
  6. implement 可以实现多接口
  7. 必须重写接口中的方法

内部类

成员内部类

Outer outer = new Outer();

Outer.Inner inner = outer.new Inner();

可获得外部类的私有属性

静态内部类

无法访问外部类的属性和方法

匿名内部类

一个Java类中可以有多个class类，只能有一个public class类

new student().study();

不用将实例保存在变量中
